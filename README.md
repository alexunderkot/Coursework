# Введение

Задачи теории расписаний относятся к числу наиболее практически значимых задач комбинаторной оптимизации. Одной из классических постановок является конвейерная задача (задача Джонсона): требуется найти такой порядок выполнения n заявок на m обслуживающих приборах, при котором суммарное время выполнения всех работ минимально. Данная задача относится к классу NP-трудных при m ≥ 3, что обусловливает применение эвристических и метаэвристических методов оптимизации.

Целью данной курсовой работы является:

1. Изучить постановку конвейерной задачи Джонсона и её математическую модель.
2. Реализовать алгоритмы получения начального решения: случайный порядок, жадный алгоритм и метод Hill Climbing.
3. Реализовать эволюционный генетический алгоритм (ЭГА) для решения задачи с операторами селекции, скрещивания, мутации и управления популяцией.
4. Провести вычислительный эксперимент и оценить относительное отклонение полученных решений от рекордных значений.
5. Построить диаграмму Ганта для визуализации найденного расписания.

Актуальность темы обусловлена широким применением задач планирования производства в промышленности, логистике и управлении проектами. Задача Джонсона является модельной для целого класса реальных задач оперативного планирования [1].

# 1. Постановка задачи

## 1.1. Содержательное описание проблемы

Конвейерная задача формулируется следующим образом. Имеется n заявок (работ) и m обслуживающих приборов (станков). Каждая заявка должна пройти обработку последовательно на всех приборах, начиная с первого. Все заявки поступают на обслуживание одновременно. Для всех заявок известны времена их обработки на каждом приборе. Прерывание процесса обслуживания заявки невозможно. Предполагается, что все приборы обслуживают заявки в одном и том же порядке. Требуется построить оптимальное по быстродействию расписание [2].

В данной работе рассматривается задача размерности 15 работ и 5 станков (15×5). Данные загружаются из стандартного набора тестовых задач с известными рекордными решениями.

## 1.2. Математическая модель

Исходные параметры модели:

- n — количество заявок; в данной работе n = 15;
- m — количество приборов (станков); m = 5;
- T = {t_ij}_{n×m} — матрица времён выполнения заявок на приборах, где t_ij — время выполнения заявки i на приборе j.

Варьируемые параметры:

- X = {x_ij}_{n×m} — матрица времён начала выполнения заявок на приборах, где x_ij — время начала выполнения i-й заявки на приборе j;
- π — перестановка заявок, определяющая порядок их выполнения.

Ограничения математической модели:

1. x_ij ≥ x_kj + t_kj или x_kj ≥ x_ij + t_ij, i,k=1..n, j=1..m — на одном приборе одновременно может выполняться только одна заявка;
2. x_{i,j+1} ≥ x_ij + t_ij, i=1..n, j=1..m-1 — выполнение работы на следующей машине может начаться только после окончания её выполнения на предыдущей;
3. если x_ij ≥ x_kj, то x_is ≥ x_ks, i,k=1..n, j,s=1..m — порядок выполнения работ на всех машинах одинаковый;
4. x_ij ≥ 0, i=1..n, j=1..m — естественные ограничения.

## 1.3. Постановка задачи

Критерием оптимальности является минимизация суммарного опоздания (Total Tardiness):

*F(π) = Σ max(0, C_i − d_i) → min*

где C_i — момент завершения выполнения i-й заявки на последнем приборе; d_i — дедлайн (срок выполнения) i-й заявки.

Дополнительные обязательные ограничения: первая работа в расписании фиксирована (позиция 0), последняя работа фиксирована (позиция 14). Эти ограничения реализованы в классе Constraints через три метода: жёсткое восстановление (MyConstraint), штрафная функция (FineConstraint) и элиминация недопустимых особей (EliminateConstraint).

Алгоритмическая сложность полного перебора составляет O(n!), что при n=15 даёт 1 307 674 368 000 вариантов — точное решение практически недостижимо, поэтому применяются эвристические и метаэвристические методы.

# 2. Подходы к решению задачи

## 2.1. Обзор существующих методов

Для решения конвейерной задачи применяются следующие методы [3]:

**Точные алгоритмы:**

- Полный перебор — гарантирует оптимум, но имеет экспоненциальную сложность O(n!) и применим только для малых n;
- Метод ветвей и границ — значительно сокращает перебор за счёт отсечения бесперспективных ветвей;
- Динамическое программирование — применимо при специальных структурах задачи [4].

**Эвристические алгоритмы:**

- Генетические алгоритмы — моделируют процессы естественного отбора [5];
- Имитация отжига — основана на аналогии с физическим процессом охлаждения вещества;
- Муравьиные алгоритмы — используют коллективный интеллект роя;
- Hill Climbing — итеративный локальный поиск в окрестности текущего решения [6];
- Жадные алгоритмы — строят решение пошагово, выбирая локально оптимальный шаг.

## 2.2. Алгоритмы получения начального решения

В работе реализованы три способа формирования начальной перестановки:

Случайная генерация (RandomOrder). Каждая работа выбирается случайным образом без повторений. Реализовано через перемешивание массива индексов методом random.Shuffle(). Данный метод быстрый, но не гарантирует качественного стартового решения.

Жадный алгоритм (GreedyOrder). Работы выбираются последовательно с вероятностью, обратно пропорциональной их дедлайну (метод рулеточного выбора). Работы с более ранними сроками получают больший вес. Это позволяет получить стартовую перестановку, лучше ориентированную на минимизацию опоздания.

Управляемая случайная генерация (ControlRandomOrder). Начинается со случайной перестановки, затем применяется цикл попарных улучшений: каждая пара соседних элементов проверяется на возможность улучшения при обмене с двумя случайными позициями. Это гибридный метод, сочетающий случайность и локальный поиск.

## 2.3. Метод Hill Climbing

Метод восхождения на холм (Hill Climbing) стартует с некоторого начального решения и итеративно исследует его окрестность. Если в окрестности найдено лучшее решение, оно принимается в качестве текущего. Процесс повторяется до тех пор, пока в окрестности не останется решений, лучших по значению целевой функции (достигнут локальный оптимум).

В данной работе окрестность определяется как множество всех перестановок, получаемых одним обменом (swap) двух элементов исходной перестановки. Сложность одной итерации — O(n²). Метод реализован в функции HillClimbingOrder() класса Individual.

## 2.4. Эволюционный генетический алгоритм (ЭГА)

ЭГА реализует следующую схему:

1. Инициализация популяции особей (перестановок) с вычислением значения целевой функции (суммарное опоздание Late) и приспособленности Fitness = 1/(Late + 1).
2. Селекция родителей: рулеточная (RouletteSelection), турнирная (TournamentSelection), ранговая (RankSelection).
3. Выбор пар родителей: случайный (RandomParents), положительный ассортативный (PositiveAssortParents), отрицательный ассортативный (NegativeAssortParents).
4. Скрещивание: операторы OX (Order Crossover), PMX (Partially Mapped Crossover), PBX (Position-Based Crossover).
5. Мутации: обменная (SwapMutation), инверсионная (InversionMutation), перемешивание (ScrambleMutation).
6. Управление популяцией: (μ+λ)-стратегия (MuNLambda) — объединение родителей и потомков с отбором лучших; (μ,λ)-стратегия (NoMuLambda) — потомки полностью заменяют родителей.
7. Обработка ограничений: жёсткое восстановление, штрафная функция или элиминация особей, нарушающих фиксированные позиции.

Программа реализована на языке C# (.NET) в объектно-ориентированном стиле. Код приведён в Приложении 1.

Система позволяет:

1. Загружать исходные данные задач из файла стандартного формата (матрицы времён обработки, рекордные значения);
2. Формировать начальную популяцию: случайно, жадным алгоритмом, методом Hill Climbing или их комбинацией;
3. Выполнять заданное количество итераций ЭГА с настраиваемыми операторами;
4. Отслеживать динамику лучшего решения по поколениям;
5. Строить диаграмму Ганта в формате HTML с автоматическим открытием в браузере;
6. Вычислять относительное отклонение от рекордного значения: δ = (F_прибл − F₀) / F₀ × 100%.

## 3.2. Пример решения задачи

Рассмотрим пример решения задачи размерности 15 работ × 5 станков. Данные загружаются из файла, содержащего несколько тестовых экземпляров. Для каждого экземпляра выполняется следующая последовательность:

1. Загрузка данных: считываются времена обработки t_ij для 15 работ на 5 станках, известное рекордное значение F₀ (Upper Bound).
2. Инициализация популяции: формируется популяция из N особей. Дедлайны вычисляются пропорционально UB: d_k = UB × k / n.
3. Запуск ЭГА: выполняется заданное число поколений. На каждом шаге применяются: селекция → выбор пар → скрещивание → мутация → применение ограничений → смена поколения.
4. Вывод результата: лучшая перестановка, суммарное опоздание Late, значение Cmax (момент завершения последней работы), отклонение от рекорда δ.
5. Диаграмма Ганта: генерируется gantt.html, отображающий расписание выполнения всех 15 работ на 5 станках в виде цветных блоков.

Построение расписания выполняется методом LateCalculate() класса Individual. Матрица времён завершения jobTime[j][i] вычисляется по формуле:

*jobTime[j][i] = max(jobTime[j][i-1], jobTime[j-1][i]) + t[j][i]*

Суммарное опоздание вычисляется как сумма положительных отклонений времён завершения от дедлайнов: Late = Σ max(0, jobTime[j][m-1] − d[j]).

## Individual.cs (фрагмент — LateCalculate)

```csharp
public int LateCalculate()
{
    int n = Data.NumJobs, m = Data.NumMachines, sum = 0;
    List<List<int>> jobTime = new();
    List<List<int>> sortedArr = SortInOrder();
    List<int> sortedDeadline = new();
    foreach (int o in Order) sortedDeadline.Add(Data.deadline[o]);
    for (int i = 0; i < n; i++)
        jobTime.Add(Enumerable.Repeat(0, m).ToList());

    jobTime[0][0] = sortedArr[0][0];
    for (int j = 1; j < n; j++)
        jobTime[j][0] = sortedArr[j][0] + jobTime[j - 1][0];
    for (int i = 1; i < m; i++)
        jobTime[0][i] = sortedArr[0][i] + jobTime[0][i - 1];
    for (int j = 1; j < n; j++)
        for (int i = 1; i < m; i++)
            jobTime[j][i] = Math.Max(jobTime[j][i-1],
                            jobTime[j-1][i]) + sortedArr[j][i];

    for (int j = 1; j < n; j++)
        sum += Math.Max(0, jobTime[j][m-1] - sortedDeadline[j]);
    return sum;
}
